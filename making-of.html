Décisions

Décisions techniques critiques

💰 Argent en centimes (INTEGER), jamais FLOAT

Toutes les colonnes monétaires sont en amount_cents INTEGER. Les floats introduisent des erreurs d'arrondi (ex. 0.1 + 0.2 ≠ 0.3 en IEEE 754). La vue v_user_monthly_cost normalise toutes les périodes en mensuel équivalent.

🔄 hamon_eligible calculé par trigger, pas par l'app

Le trigger fn_compute_hamon calcule automatiquement hamon_eligible_date = start_date + 12 mois et hamon_eligible = (hamon_eligible_date <= CURRENT_DATE) à chaque INSERT/UPDATE. Le job pg_cron rafraîchit chaque nuit.

📅 tacit_renewal_date : saisie manuelle uniquement

Ne jamais auto-calculer depuis start_date. La date de reconduction tacite dépend des termes du contrat opérateur. L'utilisateur la saisit ou elle sera scrapée en V2. Le trigger crée l'alerte selon les préférences de notification.

🏠 FK circulaire households ↔ profiles

profiles.household_id → households.id et households.created_by → profiles.id. Solution : créer le household FIRST, puis mettre à jour profiles.household_id dans une fonction SECURITY DEFINER atomique.

⚖️ Mandats non-auto-renouvelables (Art. 1301 C.civ)

Contrainte expiry_date <= effective_date + 1 an en base. Pas de champ auto_renew. Le pg_cron alerte le mandant J-30. En V2 : signature électronique qualifiée via CertEurope/DocuSign.

🔐 Commission toujours divulguée (contrainte DB)

La table recommendations a une contrainte CHECK : commission_type = 'none' OR commission_disclosed = TRUE. Il est impossible d'insérer une recommandation avec commission non déclarée. Conformité DGCCRF/AMF.

📊 features JSONB sur platform_plans

Les feature flags sont stockés en JSONB {"hamon_letter": true, "mandate_actif": false, ...}. Ajouter une feature ne nécessite aucune migration de schéma — juste un UPDATE sur la ligne du plan concerné.

🗑 Soft-delete + RGPD (Art. 17)

Soft-delete sur profiles.deleted_at cascade via trigger sur subscriptions et documents. audit_logs et rgpd_consent_logs sont exemptés (obligation légale de conservation). Après 30j : job pg_cron anonymise les colonnes PII.

pg_cron — Jobs nocturnes (02h UTC)

🌙 nightly-hamon-refresh (02:00)

Rafraîchit hamon_eligible sur tous les abonnements actifs dont la date d'éligibilité est passée.

🌙 nightly-expiry-status-refresh (02:30)

Recalcule expiry_status sur tous les documents non-supprimés (ok/warn/bad/expired).

🌙 expire-pending-invitations (02:15)

Passe status = 'expired' sur les invitations pending dont invitation_expires_at < now().

🌙 mandate-expiry-alerts (02:45)

Crée des alertes renouvellement pour les mandants dont le mandat expire dans 30 jours.
